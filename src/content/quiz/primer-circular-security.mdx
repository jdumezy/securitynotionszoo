explanation: "Mainstream FHE schemes (BFV, BGV, CKKS, TFHE) must publish evaluation keys that encrypt the secret key (or $sk \\otimes sk$, or Galois automorphisms of $sk$) under its own public key. Standard IND-CPA says nothing about the security of such key-dependent encryptions, so circular (key-dependent message) security is an extra assumption required for practical FHE. No unconditional proof reduces it to standard lattice assumptions like LWE; it is taken as a heuristic."
