Deterministic encryption does not use randomness during the encryption process. 
As a result, an adversary can easily notice if the same message is encrypted multiple times.

The IND-CPA-D security game assumes the adversary does not query the encryption oracle on a message they intend to challenge.
For deterministic encryption, IND-CPA is impossible to achieve typically because the adversary could just re-encrypt the challenge plaintexts using the public key and compare them to the challenge ciphertext.
